# skills一覧.md のズレ（たとえ話）

**結論:** スキル本体の不具合じゃなく、`skills一覧.md` の**目次・件数・置き場の説明が食い違ってる**話。

---

## たとえ

本棚に本が94冊ある。  
カタログには「デザイン棚に8冊」と書いてあるのに、中身を数えると6冊＋参照札が混ざってる。  
本は足りてる。**カタログの書き方がズレてる**だけ。

---

## ズレ① 同じスキルが複数行ある

一覧は「全94件」なのに、表の行をそのまま数えると **101行前後** になる。

理由は「参照行」:

| 置き場所 | 書いてあること |
|----------|----------------|
| 6章 | `/git-guardrails`（5章参照） |
| 7章 | `/content-research-writer`（6章参照） |
| 9章 | `/edit-article`（6章参照）…など |

正本は1つだけなのに、**別の章にもう一度名前が出る**。  
人が読む分には便利だが、Excel に落とすと「101件ある」ように見える。

---

## ズレ② スキルの置き棚がおかしい

| スキル | 今の章 | 直感的な棚 |
|--------|--------|------------|
| `/git-in-clone` | 6 デザイン | 8 GitHub/Git |
| `/git-guardrails` | 5 に正本＋6 にも出る | Git 系なら 8 に寄せた方が分かりやすい |

「デザイン・コンテンツ」の見出しなのに Git が入ってる、というズレ。

---

## ズレ③ 見出しの数字と中身が合わない

| 章 | 見出しの数字 | 実態（ざっくり） |
|----|--------------|------------------|
| §6 | 8件 | 参照や誤配置を除くと実質もっと少ない |
| §7 | 3件 | 正本はほぼ2件（1件は参照） |
| §8 | 3件 | 正本が章をまたいで散らばってる |

「足したら94」と書いてあるのに、章ごとの数字で検算しにくい。

---

## 何が困るか / 困らないか

| | |
|--|--|
| **困らない** | Cursor でスキルが動くこと自体（本体は `skills-pack`） |
| **困る** | Excel/PDF の件数、章の説明、「94件の内訳」を人に説明するとき |

「同じ `/xxx` が3つ並ぶ」は **Cursor の読み込み設定**の話。  
このズレは **一覧ドキュメントの整理**の話。別問題。

---

## 直すなら何をするか（イメージだけ）

1. 各スキルは表に **1行だけ**（参照行は消すか、章末の一文にする）
2. Git 系は Git の章に寄せる
3. 章の「○件」を、実際のユニーク数と一致させる
