# 引き継ぎ書（別PCセットアップ）

**人間のヒデさん向け:** このファイルがあるフォルダを Cursor のエージェントに見せて、下の「コピペ用の一言」をそのまま送るだけでOK。

**エージェント向け:** ユーザーがコードを書けない前提。このファイルの手順を**全部自分で実行**すること。説明だけで終わらせない。

---

## コピペ用の一言（これをそのまま送る）

```
@skills-pack/引き継ぎ.md に書いてある手順どおりに、このPCの Cursor グローバルスキルとフックのセットアップを全部やって。install.ps1（Windows）または install.sh（Mac）を実行して、既存スキルと重複しないようにマージして、最後に確認結果を教えて。
```

フォルダを添付できないとき:

```
skills-maker リポジトリの skills-pack/引き継ぎ.md に従って、このPCの Cursor グローバルスキルとフックを全部セットアップして。既存スキルと同名のものは中身が同じならスキップ、重複パスは skills-pack 版を正として整理して。
```

---

## エージェント実行手順（必ずこの順番）

### 1. 場所を確認

- このファイルは `skills-maker/skills-pack/引き継ぎ.md` にある
- 同じフォルダに `install.ps1`（Windows）と `install.sh`（Mac/Linux）がある
- リポジトリのルートがわからなければ、ユーザーに `skills-maker` のパスを聞く

### 2. 最新版を取得（clone 済みなら）

```powershell
cd <skills-maker のパス>
git pull
```

`git pull` が失敗したら理由を報告し、続行可否をユーザーに確認。

### 3. インストールを実行（手コピー禁止）

**Windows（PowerShell）:**

```powershell
cd <skills-maker のパス>\skills-pack
.\install.ps1
```

**macOS / Linux:**

```bash
cd <skills-maker のパス>/skills-pack
chmod +x install.sh
./install.sh
```

- スキルフォルダを1個ずつ手でコピーしない
- `/add-plugin superpowers` は使わない（TDD / writing-plans が上書きされる）
- `install.ps1` / `install.sh` だけが正しい方法
- **重複処理はインストーラ内蔵**（同名・同内容はスキップ、旧ネスト構造は削除、正本は skills-pack のパス）

### 4. インストール後の確認（エージェントが実行して報告）

次が存在するか確認:

| 確認項目 | Windows の例 |
|----------|----------------|
| スキル数 | `MANIFEST.json` に 94 件 |
| スキル | `%USERPROFILE%\.cursor\skills\playbooks\playbook-document-data\SKILL.md` |
| スキル | `%USERPROFILE%\.cursor\skills\gws-docs\SKILL.md` |
| フック | `%USERPROFILE%\.cursor\hooks\session-start.ps1`（Windows） |
| 設定 | `%USERPROFILE%\.cursor\hooks.json` に `sessionStart` がある |
| 重複 | 同名スキル `name:` が 2 件以上ない |

重複が残る場合は [skills重複処理.md](skills重複処理.md) を実行。

macOS/Linux では `~/.cursor/skills/` と `~/.cursor/hooks/`。

### 5. ユーザーへの報告（日本語・やさしく）

次を必ず伝える:

1. **セットアップ完了した**
2. **Cursor を一度終了して起動し直す**
3. **Customize → Skills** で `/playbook-document-data` や `/gws-docs` などが見えるか確認（`/writing-plans` が2件なら [skills重複処理.md](skills重複処理.md) を実行）
4. **Customize → Hooks** で `session-start` があるか確認
5. Google 連携を使う場合は **`gws auth login` が別 PC でも必要** と伝える
6. 問題があれば `skills-pack/INSTALL.md` のトラブルシューティングを参照

## 注意（別PC向け・必読）

- **GWS OAuth（`gws auth login`）は PC ごと。** skills-pack には OAuth 設定は入らない。`playbook-document-data` や `gws-*` スキルで Google Docs / Sheets / Drive を使う場合は、別 PC でも `gws` CLI のインストールと OAuth セットアップが必要。
- **別 PC に Superpowers プラグインが入っていると重複の可能性あり。** `install.ps1` 実行後、Customize → Skills で `/writing-plans` などが **2件** 出たら [skills重複処理.md](skills重複処理.md) をエージェントに実行させる。`/add-plugin superpowers` は使わないこと。
- **同じ PC で Claude Code も使う場合:** Cursor が `~/.claude/skills/` も読んで同名が 2〜3 件並ぶ。`~/.claude` は消さず、Cursor Settings → Rules, Skills, Subagents → **Include Third-Party Plugins, Skills, and Other Configs** を OFF。詳細は [skills重複処理.md](skills重複処理.md)。

---

## やってはいけないこと

- スキルを1個ずつ手動コピーする
- Superpowers プラグインを丸ごと入れる
- `test-driven-development` と `writing-plans` を Superpowers 版で上書きする
- `~/.cursor/skills-cursor/` を触る
- セットアップ完了を報告する前に確認コマンドをスキップする
