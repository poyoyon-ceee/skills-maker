# 引き継ぎ書（別PCセットアップ）

**人間のヒデさん向け:** このファイルがあるフォルダを Cursor のエージェントに見せて、下の「コピペ用の一言」をそのまま送るだけでOK。

**エージェント向け:** ユーザーがコードを書けない前提。このファイルの手順を**全部自分で実行**すること。説明だけで終わらせない。

---

## コピペ用の一言（これをそのまま送る）

```
@skills/引き継ぎ.md に書いてある手順どおりに、このPCの Cursor グローバルスキルとフックのセットアップを全部やって。install.ps1（Windows）または install.sh（Mac）を実行して、最後に確認結果を教えて。
```

フォルダを添付できないとき:

```
skills-maker リポジトリの skills/引き継ぎ.md に従って、このPCの Cursor グローバルスキルとフックを全部セットアップして。
```

---

## エージェント実行手順（必ずこの順番）

### 1. 場所を確認

- このファイルは `skills-maker/skills/引き継ぎ.md` にある
- 同じフォルダに `install.ps1`（Windows）と `install.sh`（Mac/Linux）がある
- リポジトリのルートがわからなければ、ユーザーに `skills-maker` のパスを聞く

### 2. 最新版を取得（clone 済みなら）

```powershell
cd <skills-maker のパス>
git pull
```

`git pull` が失敗したら理由を報告し、続行可否をユーザーに確認。

### 3. インストールを実行（手コピー禁止）

**Windows（PowerShell）:**

```powershell
cd <skills-maker のパス>\skills
.\install.ps1
```

**macOS / Linux:**

```bash
cd <skills-maker のパス>/skills
chmod +x install.sh
./install.sh
```

- スキルフォルダを1個ずつ手でコピーしない
- `/add-plugin superpowers` は使わない（TDD / writing-plans が上書きされる）
- `install.ps1` / `install.sh` だけが正しい方法

### 4. インストール後の確認（エージェントが実行して報告）

次が存在するか確認:

| 確認項目 | Windows の例 |
|----------|----------------|
| スキル | `%USERPROFILE%\.cursor\skills\writing-plans\SKILL.md` |
| スキル | `%USERPROFILE%\.cursor\skills\superpowers\brainstorming\SKILL.md` |
| フック | `%USERPROFILE%\.cursor\skills\` ではなく `%USERPROFILE%\.cursor\hooks\session-start.ps1`（Windows） |
| 設定 | `%USERPROFILE%\.cursor\hooks.json` に `sessionStart` がある |

macOS/Linux では `~/.cursor/skills/` と `~/.cursor/hooks/`。

### 5. ユーザーへの報告（日本語・やさしく）

次を必ず伝える:

1. **セットアップ完了した**
2. **Cursor を一度終了して起動し直す**
3. **Customize → Skills** で `/writing-plans` などが見えるか確認
4. **Customize → Hooks** で `session-start` があるか確認
5. 問題があれば `skills/INSTALL.md` のトラブルシューティングを参照

## やってはいけないこと

- スキルを1個ずつ手動コピーする
- Superpowers プラグインを丸ごと入れる
- `test-driven-development` と `writing-plans` を Superpowers 版で上書きする
- セットアップ完了を報告する前に確認コマンドをスキップする
